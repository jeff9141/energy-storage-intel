# MTS 与家储系统连接拓扑

## 概述

MTS（Manual Transfer Switch，手动转换开关）是家庭储能系统中实现停电备份的关键安全器件。它通过物理隔离电网与逆变器输出，满足 anti-islanding（防孤岛）安全要求，防止逆变器在停电时向电网反送电，保护电网抢修人员安全。

---

## 系统组成

| 组件 | 功能 |
|------|------|
| **电网（Grid）** | 市电供电 |
| **电表（Meter）** | 计量用电 |
| **MTS** | 双掷开关，切换电网/逆变器供电 |
| **家庭配电箱（Main Panel）** | 分配电力到各回路 |
| **混合逆变器（Hybrid Inverter）** | DC/AC 转换，管理并网/离网切换 |
| **电池组（Battery Pack）** | 储存电能 |
| **光伏组件（PV Array）** | 发电（可选） |

---

## 连接拓扑

### 物理接线

系统有 **4 条关键线路**：

```
① 电网 → 电表 → MTS（Line 输入端）
② MTS（Load 输出端） → 家庭配电箱 → 家庭负载
③ 配电箱（专用断路器） ↔ 逆变器 AC IN 端口（并网口，双向）
④ 逆变器 AC OUT 端口（备用口） → MTS（Backup 输入端）
```

## 工作场景

### 场景一：电网正常（MTS 在电网位）

`═══` 表示有电流通过，`- - -` 表示断开/空载。

```
  ┌──────┐       ┌──────────────┐       ┌───────────┐
  │ 电网  ├═══════╡     MTS      ╞═══════╡  家庭配电箱 ╞═══→ 负载
  │ Grid │       │   [电网位]    │       │ Main Panel│
  └──────┘       │              │       └─────╤═════┘
                 │  Backup 输入  │             ║
                 └──────┬───────┘        ③ AC IN（双向）
                        ╎                     ║
                   ④ AC OUT              ┌════╧═══════════┐
                    (空载)               │    混合逆变器     │
                        ╎               │ Hybrid Inverter │
                        ╎               └══╤═════════╤════┘
                     - - -                 ║         ║
                     断 开               电池组    PV 光伏
                                        (充放电)   (发电)
```

**此场景下的能量流向：**

| 功能 | 电流路径 | 状态 |
|------|---------|------|
| 电网供电 | 电网 → 电表 → MTS(Line) → 配电箱 → 负载 | **通** |
| 光伏自发自用 | PV → 逆变器 → AC IN → 配电箱 → 负载 | **通** |
| 市电给电池充电 | 电网 → 配电箱 → AC IN → 逆变器 → 电池 | **通** |
| 电池放电自用 | 电池 → 逆变器 → AC IN → 配电箱 → 负载 | **通** |
| 馈网卖电 | PV/电池 → 逆变器 → AC IN → 配电箱 → MTS → 电网 | **通** |
| AC OUT 供电 | 逆变器 → AC OUT → MTS Backup 输入 | **断** |

### 场景二：电网停电（手动切换 MTS 至备用位）

```
  ┌──────┐       ┌──────────────┐       ┌───────────┐
  │ 电网  ├╌╌╌╌╌╌╡     MTS      ╞═══════╡  家庭配电箱 ╞═══→ 负载
  │✕ 停电│ 断开  │   [备用位]    │       │ Main Panel│
  └──────┘       │              │       └───────────┘
                 │  Backup 输入  │
                 └══════╤═══════┘
                        ║
                   ④ AC OUT（供电中）
                        ║
                 ┌══════╧═════════════┐
                 │     混合逆变器       │
                 │  [离网模式 V/f 源]   │
                 └══╤═════════╤═══════┘
                    ║         ║
                  电池组    PV 光伏
                 (放电中)   (发电中)
```

**操作流程：**

1. 用户发现停电（或收到 App 推送）
2. **手动拨动 MTS 至「备用位」** — 物理断开电网，接通逆变器 AC OUT
3. 逆变器检测到与电网断开，自动切换为 **离网模式（V/f 源）**
4. 电池开始放电，通过 AC OUT → MTS → 配电箱 → 给家庭负载供电
5. 白天光伏继续发电，给电池充电 + 直接供负载
6. 电网恢复后，手动将 MTS 拨回「电网位」，逆变器恢复并网模式

## 核心概念：AC IN 是双向端口

!!! warning "AC IN / AC OUT 命名容易误导"
    很多人直觉认为 AC IN 只能「进」、AC OUT 只能「出」——这是错误的。

混合逆变器的两个 AC 端口本质：

| 端口 | 实际含义 | 方向 | 主要用途 |
|------|---------|------|---------|
| **AC IN** | Grid Port（并网口） | **双向** | 并网运行时的所有能量交互：充电、放电自用、馈网 |
| **AC OUT** | Backup / EPS Port（离网口） | 输出为主 | 停电时给备用负载供电 |

命名来源是**停电场景**的视角：停电时电从 AC OUT「出」去供电，AC IN 因为没有电网所以没有电「进」来。但在正常并网运行时，AC IN 是主力端口，**进出都走它**。

### 市电充电路径（AC IN 正向）

```
电网 → 电表 → MTS(电网位) → 配电箱 → 断路器 → 逆变器 AC IN → DC转换 → 电池充电
                                                   ──────────→
```

### 电池自发自用路径（AC IN 反向）

```
电池放电 → 逆变器 DC→AC → AC IN(反向输出) → 断路器 → 配电箱 → 家庭负载消纳
                          ←──────────────
```

逆变器在并网模式下与电网同步（频率/电压跟随电网），通过 AC IN 端口向配电箱「推」电流。配电箱上的负载优先消纳逆变器输出的电，不够的部分从电网补。

## MTS vs ATS 对比

| 特性 | MTS（手动转换开关） | ATS（自动转换开关） |
|------|-------------------|-------------------|
| **切换方式** | 人工手动拨动 | 逆变器/控制器自动切换 |
| **切换时间** | 分钟级（取决于人的反应） | 毫秒级（< 20ms） |
| **用户体验** | 停电后需要人工操作 | 无感切换，UPS 级体验 |
| **成本** | 低（$100–300） | 高（内置于逆变器或独立模块） |
| **合规复杂度** | 低，物理隔离天然满足 anti-islanding | 高，需要通过 UL1741 等认证 |
| **典型产品** | 低成本户储方案（Bluetti、EcoFlow 等） | 高端全屋备份（Tesla Powerwall、Enphase IQ） |
| **适用场景** | 预算敏感、停电不频繁、可接受手动操作 | 追求无感体验、停电频繁、关键负载不可中断 |

## 关键安全要求

!!! danger "Anti-Islanding（防孤岛）"
    MTS 存在的根本原因是 **防止逆变器在停电时向电网反送电**。如果不断开电网侧，逆变器输出可能沿线路反送到配电网，对正在抢修的电力工人造成致命触电危险。MTS 通过物理隔离保证「要么接电网，要么接逆变器」，两者绝不同时导通。这是 NEC（美国国家电气规范）和 UL 标准的硬性合规要求。
