> 最后更新：2026-06-14
> 关联主文档：[储能行业政策法规汇总](./energy_storage_policy_regulations.md) > 一、欧盟层面 > 1.1

## 一、法规概述

EU 电池法规（Regulation 2023/1542）对碳足迹的要求按**"申报→分级→准入门槛"三阶段递进**实施，是全球首个覆盖电池全生命周期的强制性碳足迹法规。

### 1.1 三阶段时间线

```
阶段一：碳足迹申报         阶段二：性能等级标签         阶段三：准入上限
(Declaration)              (Performance Class)          (Maximum Threshold)
        │                          │                          │
   ┌────┴────┐               ┌─────┴─────┐             ┌─────┴─────┐
   │ 告知义务 │               │ 等级分类  │             │ 市场禁入  │
   │"你排了多少"│             │"你排在哪档"│            │"超标不准卖"│
   └─────────┘               └───────────┘             └───────────┘
```

**各电池品类适用时间线：**

| 阶段 | EV 电池 | >2 kWh 工业电池（家储/阳储） | LMT 电池（部分移储） |
|------|---------|---------------------------|-------------------|
| 阶段一：碳足迹申报 | 2025-02-18 | **2026-02-18** | 2028-02-18 |
| 阶段二：性能等级标签 | 2026-02-18 | **2026-08-18** | 2028-08-18 |
| 阶段三：准入上限 | 2027-07（委托法规） | 待定（预计 2028+） | 待定 |

### 1.2 申报技术要求

| 维度 | 要求 |
|------|------|
| **方法论** | PEF（Product Environmental Footprint），EU 自有方法论 |
| **标准依据** | ISO 14040/44（LCA 框架）、ISO 14067（碳足迹量化）、EU PEFCR |
| **度量单位** | kg CO₂e / kWh（全生命周期交付能量的二氧化碳当量） |
| **LCA 边界** | 原材料开采 → 活性材料制造 → 电芯制造 → 电池组装 → 分销 → 报废处理 |
| **申报粒度** | Per 电池型号 Per 制造工厂 |
| **验证要求** | 必须经第三方验证 |
| **格式规范** | 欧盟委员会于 `[2024-04]` 发布委托法规和实施法规，规定标准化模板 |

### 1.3 性能等级标签

- 类似家电能效标签，A/B/C/D… 分级，A 级代表最低碳足迹
- 等级划分基于市场已申报碳足迹的分布基准
- 标签必须**清晰、可读、永久标注**在电池本体上（不是纸质标签）

### 1.4 准入上限

- 欧盟委员会将设定每种电池类型的**最高允许碳足迹值**
- **超过上限的电池将被禁止在 EU 市场销售** — 这是硬性市场准入门槛
- EV 电池的上限将于 2027-07 通过委托法规公布，工业电池预计 2028 后跟进

### 1.5 数字电池护照

| 要求 | 说明 |
|------|------|
| **适用范围** | EV 电池 + >2 kWh 可充电工业电池 `[家储]` `[阳储]` |
| **生效时间** | 2027-02-18 |
| **唯一标识** | 每块电池分配唯一标识码（Unique Identifier） |
| **QR 码** | 必须在电池本体上印刷或刻蚀 QR 码，链接至数字护照 |
| **数据可用性** | 电池整个服务寿命期间保持准确、安全、可访问（10-15 年+） |
| **数据模型** | 基于 AAS（Asset Administration Shell）和 SAMM 标准 |
| **技术生态** | Battery Pass 联盟 + GBA + Catena-X 数据生态 |
| **数据分层** | 公开数据（碳足迹、成分、可回收性）vs 受控数据（IP 相关制造细节） |
| **开源参考** | GitHub: batterypass/BatteryPassDataModel |

## 二、不做的后果 — 四级影响链路

### 2.1 第一级：硬性禁入（法律后果）

碳足迹申报是 CE 认证的前置条件。

```
无碳足迹申报 → 无法完成 CE 认证 → 产品不得在 EU 27 国 + EEA 销售
```

- 海关直接扣押，市场监管部门处罚
- 电商平台（Amazon DE、Kaufland 等）强制下架
- 2027-02 起，无 QR 码和数字护照的电池同样被禁止销售

**这不是罚款了事，是物理层面无法进入市场。**

### 2.2 第二级：渠道封锁（比法规生效更早显现）

合规文件不完整的影响会在法规生效日之前就在业务层面体现：

| 渠道 | 影响机制 |
|------|---------|
| **安装商** | 帮用户申请 KfW/Ecobonus 等补贴需提交产品合规文件，文件不全 → 安装商直接换品牌 |
| **分销商** | Krannich、BayWa r.e. 等欧洲主力分销商在选品阶段就要求供应商提供完整合规文档包 |
| **VPP 聚合商** | Sonnen/Tesla VPP 网络要求接入设备合规，不合规设备无法接入 = 丧失 VPP 收入场景 |
| **电商平台** | Amazon、MediaMarkt 等平台逐步要求卖家提供 CE/EPR 合规证明，否则限制上架 |

### 2.3 第三级：竞争降级（做了但没做好）

性能等级标签是 A/B/C/D 分级。不主动优化碳足迹而随便报一个数 → 拿到 C 级或 D 级：

- 产品能卖，但在货架上类似"五级能效空调"贴在"一级能效"旁边，消费者和安装商用脚投票
- 奥地利"Made in Europe" +10% 补贴已经建立了碳足迹与补贴挂钩的先例，未来可能有更多国家将补贴与性能等级绑定
- B 端客户（地产商、能源公司）要求供应商提供碳足迹数据用于 ESG 报告，无法提供 = 被排除在采购清单外

### 2.4 第四级：战略风险（拖延的隐性成本）

| 风险 | 说明 |
|------|------|
| **数据回溯不可能** | 碳足迹计算需要每个物料的碳排放因子。现在不向电芯供应商要数据，2027 年赶工时历史批次数据可能已经拿不到 |
| **技术债务** | 被迫赶工建的电池护照平台质量打折，而这个平台需要运行 10-15 年 |
| **供应链锁定** | 大供应商优先服务已经在做碳足迹合规的客户，后来者可能面临供应商配合度下降 |
| **全球法规扩散** | 英国、澳洲大概率跟进类似法规。现在为 EU 建的合规系统可复用为全球合规基座，越早建复用价值越大 |

### 2.5 不做的总结

| 截止时间 | 必须完成 | 不做的直接后果 |
|---------|---------|-------------|
| 2026-02（已逾期） | >2 kWh 产品碳足迹申报数据准备 | 新品无法获得 CE 认证上市 |
| 2026-08 | 性能等级标签系统 | 产品无法合法标注等级 → 下架 |
| 2027-02 | 电池护照平台上线 | 所有在售 >2 kWh 产品被强制退市 |

**结论：碳足迹合规不是"做不做"的问题，而是"什么时候做完"的问题——因为不做 = 退出 EU 市场。**

## 三、软件系统影响全景 — 六层架构分析

碳足迹合规不是"填个表"，而是需要**从供应链到云平台到终端 App 全链路贯通**的系统工程。

### 3.1 L1：供应链数据采集层（PLM/ERP 侧）

**需要做什么：**

- 从电芯和关键部件供应商处获取每个物料的碳排放因子（emission factor）
- 采集制造过程能耗数据（每条产线、每个工厂）
- 采集运输里程数据（工厂→仓库→终端市场）

**软件需求：**

| 系统 | 说明 | 建设方式 |
|------|------|---------|
| **供应商碳数据管理** | 结构化采集表单/接口，供应商提交物料级 emission factor | 短期 Excel 模板，中期系统化 |
| **PLM/ERP 扩展** | BOM 关联碳数据字段，每个物料绑定碳排放因子 | 改造现有 PLM/ERP |
| **制造能耗采集** | 对接产线 MES，获取每批次的制造能耗数据 | 改造现有 MES |

**责任归属：** 通常归属供应链/质量体系团队，但产品 PM 必须知道——如果供应链数据不到位，碳足迹申报就无法完成，产品就无法上市。

### 3.2 L2：LCA 计算引擎

**需要做什么：**

- 基于 PEF/PEFCR 方法论，计算每个电池型号在每个工厂的全生命周期碳足迹
- 输出结果：kg CO₂e/kWh
- 结果需经第三方验证

**软件选型：**

| 方案 | 适用场景 | 代表工具 |
|------|---------|---------|
| **外采 LCA 计算服务** | 中小厂商、产品线少 | 委托 TUV、SGS 等第三方咨询 |
| **行业 LCA 软件** | 中大厂商、需要持续迭代 | Sphera (前 thinkstep)、SimaPro、openLCA |
| **自建简化计算模块** | 大厂、产品线多、迭代频繁 | 基于 PEFCR 规则的内部工具 |

**PM 关注点：** LCA 计算的输入参数（BOM 配方、电芯化学体系、制造能耗、运输方式）直接影响产品定义。例如：
- LFP（磷酸铁锂）的碳足迹通常低于 NMC（三元锂），选择 LFP 可能直接决定产品获得 A 级 vs C 级
- 制造工厂的能源结构（绿电 vs 煤电）显著影响碳足迹数值
- **这意味着碳足迹等级可以反向驱动产品定义和供应链选择**

### 3.3 L3：数字电池护照平台（核心新建系统）

**这是软件产品团队最直接的交付物。**

| 子系统 | 功能 | 技术要点 |
|--------|------|---------|
| **UID 管理** | 每块电池分配唯一标识码，关联生产批次、BOM 版本、碳足迹数据 | UUID 或 GS1 标准 |
| **QR 码生成** | 生成 QR 码 → 与产线 MES 打通 → 印刻在电池本体 | 必须是印刻/印刷（非纸质标签） |
| **数据托管** | 电池护照全量数据托管，10-15 年可访问 | 高可用云服务、长期运维保障 |
| **AAS API** | 基于 Asset Administration Shell 标准的对外接口 | 供监管机构、回收商、二手交易方查询 |
| **数据分层访问控制** | 公开数据 vs 受限数据（IP 保护） | RBAC / OAuth |
| **外部生态对接** | 对接 Catena-X 数据生态、GBA Battery Pass | 标准化数据交换协议 |

**技术选型方向：**

- **方案 A**：对接 Catena-X 数据生态（欧洲主流的工业数据交换平台），使用 AAS Submodel 标准
- **方案 B**：自建 AAS-compliant 数据服务，参考 Battery Pass 联盟开源数据模型（GitHub: batterypass/BatteryPassDataModel）
- **方案 C**：采购第三方 DPP（Digital Product Passport）SaaS 平台

**关键决策点：** 自建 vs 外采。如果产品线多、出货量大，自建更合算（长期数据托管成本可控）；如果产品线少、出货量小，外采 SaaS 更快。

### 3.4 L4：BMS 固件层

**需要改造什么：**

电池护照需要**动态数据**——以下数据会随电池使用而变化，必须持续采集上报：

| 数据项 | 说明 | 采集频率 |
|--------|------|---------|
| SOH（State of Health） | 电池健康度 | 每日/每周 |
| 累计循环次数 | 充放电循环总数 | 实时 |
| 容量衰减曲线 | 实际可用容量 vs 标称容量 | 每周/每月 |
| 使用环境温度 | 影响寿命预测 | 实时 |

**核心变化：** BMS→云平台的数据通道**不再是"可选的远程监控功能"，而是合规硬性要求**。

**影响评估：**
- 已有 WiFi/4G 联网能力的产品（Tesla Powerwall、Enphase、EcoFlow）：改造成本低，主要是增加数据上报字段
- 无联网能力的入门级家储：需要评估加装通信模块（成本+BOM 变更）或设计"安装商定期手动同步"的离线方案

### 3.5 L5：云平台 / EMS 层

**需要改造什么：**

| 改造项 | 说明 |
|--------|------|
| **合规数据管理模块** | 存储维护每台设备的碳足迹申报数据、性能等级、护照全量数据 |
| **设备数据模型扩展** | 现有设备表增加合规相关字段（碳足迹值、等级、UID、护照状态） |
| **动态护照数据更新** | 接收 BMS 上报的 SOH/循环次数等动态数据，更新护照动态字段 |
| **数据保留策略** | 10-15 年以上的数据保留，影响存储成本和系统可靠性架构 |
| **对外数据接口** | 向监管机构、回收商、二手市场开放合规数据查询接口 |

**对现有平台的影响举例：**
- Growatt ShineServer、Sungrow iSolarCloud：需扩展后台数据模型
- Fox ESS FoxCloud：需新增合规数据管理页面
- EcoFlow Cloud：需为家储和阳储产品线分别对接护照数据

### 3.6 L6：终端 App / Web

**用户侧影响相对间接，但可以将合规义务转化为营销资产：**

| 功能 | 说明 | 价值 |
|------|------|------|
| **碳足迹等级展示** | "您的电池碳足迹等级：A 级" | 类似家电能效标签，建立品牌高端认知 |
| **碳减排成就可视化** | "已避免 XX kg CO₂ 排放" | 将合规义务转化为用户价值感知和分享动力 |
| **电池护照查看** | App 内 QR 扫码 / 数字护照入口 | 增强产品透明度和用户信任 |
| **合规信息透出** | CE / EPR / 性能等级标签 | 渠道和消费者信任基础 |

## 四、对不同产品线的差异化影响

### 4.1 家庭储能 `[家储]`

**全面受影响。** 市场主力产品（5-15 kWh）全部 >2 kWh，碳足迹申报+性能等级+电池护照三项全覆盖。

| 时间 | 动作 | 优先级 |
|------|------|--------|
| 2026 Q2（现在） | 完成碳足迹申报数据准备和 LCA 计算 | P0 — 已逾期 |
| 2026 Q3 | 性能等级标签系统开发和产线标注流程 | P0 |
| 2026 Q3-Q4 | 电池护照平台架构设计和开发 | P0 |
| 2027 Q1 | 平台上线、端到端联调 | P0 |

### 4.2 阳台储能 `[阳储]`

**多数产品受影响。** 主流产品 >2 kWh（Zendure AIO 2400 = 2.4 kWh，Marstek B5-Plus = 2.56 kWh），需要全套合规。

**边界案例需逐一确认：**
- Anker SOLIX Solarbank E1600 = 1.6 kWh → 可能豁免
- 容量刚好在 2 kWh 附近的产品 → 需要确认 EU 分类规则（标称容量 vs 可用容量）

**产品定义启示：** 如果阳储产品容量刚好卡在 2 kWh 边界，**是否有意将容量控制在 <2 kWh 以规避碳足迹合规成本？** 这是一个值得评估的产品定义决策——合规成本 vs 容量损失的权衡。

### 4.3 便携/移动储能 `[移储]`

**多数产品暂不受影响。** 便携储能多数 <2 kWh，碳足迹申报推迟至 2028-02-18（LMT 类别）。

**但需注意大容量便携储能：**
- EcoFlow DELTA Pro = 3.6 kWh → 可能被归入 >2 kWh 工业电池 → 需提前合规
- Bluetti AC200MAX = 2.048 kWh → 处于边界
- 多台并联使用场景：单台 <2 kWh 但系统总容量 >2 kWh 时如何界定？需法律确认

## 五、竞争格局影响

### 5.1 碳足迹合规创造新的竞争分层

| 厂商类型 | 合规能力 | 竞争位势 |
|---------|---------|---------|
| **垂直整合大厂**（Tesla、BYD） | 供应链自控，碳数据完整 | 天然优势，合规成本低 |
| **品牌+自研厂**（Sonnen、Enphase、Sigenergy） | 有自研平台，可自建护照系统 | 中等成本，需要推动供应商配合 |
| **品牌+ODM 厂**（部分中国品牌） | 碳数据依赖 ODM 供应商提供 | 风险最高，数据断层可能导致合规延迟 |
| **纯贸易商/渠道商** | 无自研能力，完全依赖上游 | 被动等待，可能被淘汰 |

### 5.2 性能等级标签将成为新的营销战场

- A 级标签 = 低碳足迹 = 渠道和消费者心智中的高端定位
- 这对产品定义有倒推作用：**选择低碳足迹的电芯和材料，可能比追求极致能量密度更影响产品在 EU 市场的竞争位势**
- LFP 电芯碳足迹通常低于 NMC，LFP 路线的厂商在碳足迹等级上可能占优

### 5.3 合规能力成为安装商选品的新维度

安装商选品时除了看产品性能和价格，还会看：
- 合规文档是否齐全（CE + EPR + 碳足迹申报 + 性能等级）
- 电池护照系统是否正常运行（影响售后和二手市场处置）
- 品牌方是否有长期运维合规系统的能力（10-15 年数据可用性承诺）
