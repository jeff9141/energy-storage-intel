## 一、政策全景：ECO4 → Warm Homes Plan 过渡

英国家储补贴体系正在经历一次根本性重组：ECO 框架（能源公司义务）终结，Warm Homes Plan 接棒成为未来 4 年唯一的大规模住宅能效升级资金来源。

### 1.1 政策体系结构

```
英国住宅储能补贴体系（2026-2030）
│
├── Warm Homes Plan（£15B，2026-01 发布）
│   │
│   ├── Warm Homes: Local Grant（WH:LG）
│   │   适用对象：低收入私人住宅户主
│   │   资格条件：年收入 ≤£36,000 或领取福利 + EPC D-G
│   │   资金来源：政府财政拨款
│   │   补贴方式：全额补贴，含光伏+电池+热泵+隔热
│   │
│   ├── Warm Homes: Social Housing Fund（WH:SHF）— £754M
│   │   适用对象：地方政府 + Housing Association
│   │   申请方式：房东代租户申请
│   │   覆盖措施：光伏（42%）、隔热（36%）、低碳供暖（13%）
│   │   截止时间：2028-09-30 完工
│   │
│   ├── Warm Homes Fund（£2B 零息贷款）
│   │   适用对象：有支付能力的户主（不限收入）
│   │   覆盖：光伏+储能+热泵+隔热
│   │
│   └── BUS 热泵补贴（£2.7B，确认延续至 2029-30）
│       适用对象：热泵安装
│
├── ECO4（£4B，2026-12-31 终止）
│   最后的能源公司义务计划，Ofgem 监管
│   ECO5 已取消，不再延续 ECO 框架
│
├── 0% VAT（确认延续至 2027-03）
│   适用：储能硬件及安装服务
│   节省：约 £2,000（典型 4 kW 光储系统 £12,000 基准）
│
└── SEG（Smart Export Guarantee）
    余电售回电网，分时电价最高 50p/kWh
```

### 1.2 关键时间线

| 时间 | 事件 | 影响 |
|------|------|------|
| 2026-01 | Warm Homes Plan 正式发布 | £15B 总投资，替代 ECO 框架 |
| 2026-10 | EPC 改革生效 | 新增 smart readiness 指标，影响储能产品定位 |
| 2026-12-31 | ECO4 终止 | ECO 框架正式终结，不再有 ECO5 |
| 2027-03 | 0% VAT 确认有效期 | 需跟踪是否续期 |
| 2028-09-30 | WH:SHF Wave 3 完工截止 | Social Housing 改造窗口 |
| 2030-04-01 | 社会住宅 EPC C 强制达标 | Decent Homes Standard 硬性要求 |

### 1.3 储能在 WHP 中的定位

**电池储能不是独立补贴品类，是光伏的配套措施。**

| 维度 | 说明 |
|------|------|
| **补贴条件** | 必须与光伏系统一同申请，不可单独申请电池补贴 |
| **电池补贴金额** | £2,500（WH:LG 低收入户可免费获得） |
| **套装逻辑** | 典型组合：光伏 4 kW + 储能 5-10 kWh + 热泵（可选） |
| **VAT** | 0%（节省约 £2,000） |

## 二、ECO4 详解 — 终局分析

### 2.1 ECO4 核心机制

ECO（Energy Company Obligation）是英国独有的补贴机制：**不是政府直接拨款，而是强制大型能源公司出钱**。

| 维度 | 说明 |
|------|------|
| **全称** | Energy Company Obligation 4 |
| **总预算** | £4B |
| **资金来源** | 由 Ofgem 监管的大型能源供应商（客户数 >15 万的供应商）出资 |
| **适用对象** | 低收入家庭（领取 Universal Credit、Pension Credit、Child Tax Credit 等 9 种福利） |
| **覆盖措施** | 隔热（主力）、供暖系统、光伏（可含电池配套） |
| **运行周期** | 2022-04 至 2026-12-31 |
| **申请流程** | 用户联系能源供应商或授权安装商 → 资格审核 → 评估 → 安装 |

### 2.2 ECO4 的终止与 ECO5 取消

| 事件 | 说明 |
|------|------|
| **ECO4 终止** | 2026-12-31 正式结束 |
| **ECO5 原计划** | 预计 2026-04 启动，拟将电池储能纳入核心措施（E.ON 等供应商积极推动） |
| **ECO5 取消** | 政府决定终结 ECO 框架，不再以"供应商义务"模式运营 |
| **替代方案** | Warm Homes Plan 接棒，£1.5B 原 ECO 资金重新分配至 WH:SHF 和 WH:LG |

**ECO5 取消的意义：** ECO 框架下储能厂商不需要直接面对政府，只需说服能源供应商（E.ON、British Gas 等）采购自家产品。Warm Homes Plan 切换为政府拨款模式后，渠道链路变为"政府 → 地方政府 → 安装商 → 品牌"，安装商的角色进一步强化。

### 2.3 对软件产品的影响

- **ECO4 剩余窗口（至 2026-12）**：不值得为 ECO4 做专项功能开发，窗口太短
- **ECO → WHP 过渡期**：安装商同时处理 ECO4 尾单和 WHP 新单，项目管理工具的价值在过渡期最高
- **渠道链路变化**：ECO 模式下安装商是能源公司的分包商；WHP 模式下安装商直接与地方政府对接，需要 TrustMark + PAS 2030 认证

## 三、Warm Homes Plan — 私人住宅市场的软件规划

### 3.1 用户旅程与软件切入点

```

━━━━━━━━━━━━━━━━━━          ━━━━━━━━━━━━━━━━━━          ━━━━━━━━━━━━          ━━━━━━━━━━━━

"电费太贵了"                  Retrofit Assessor 上门        安装商施工              SEG 分时售电
```

### 3.2 C 端 App 功能规划

#### 3.2.1 补贴资格速查（P0）

| 维度 | 说明 |
|------|------|
| **用户场景** | "我能不能拿到补贴？能拿多少？" — 这是用户旅程的第一个问题 |
| **输入** | 邮编 + 房屋类型（flat/terrace/semi/detached）+ 收入区间 + 是否领取福利 |
| **输出** | ① 适用补贴类型（WH:LG 全额补贴 / 零息贷款 / 仅 0% VAT）② 预估补贴金额 ③ 下一步行动指引 |
| **数据源** | EPC Open Data API（英国政府开放数据，免费，按邮编/地址查询 EPC 评级） |
| **技术方案** | 规则引擎：EPC 评级（D-G 符合 WH:LG）× 收入/福利资格 × 房屋类型 → 补贴路径。无需复杂后端，前端计算即可 |

#### 3.2.2 补贴后经济账（P0）

| 维度 | 说明 |
|------|------|
| **用户场景** | "装了光伏+储能，算上补贴，多久回本？" — 转化决策的关键页面 |
| **输入** | 当前年电费（或月电费）+ 房屋朝向 + 光伏容量 + 储能容量 |
| **输出** | 系统总价 → 补贴抵扣（£2,500 电池 + 光伏部分）→ VAT 减免 → 实付金额 → 年化节省（自用省电 + SEG 售电）→ 回本年限 → 20 年总收益 |
| **计算模型** | 年化节省 = 自用量 × 购电价 + SEG 馈电量 × SEG 均价 - 系统年维护成本；回本 = 实付 / 年化节省 |
| **呈现方式** | 一页式可视化，带品牌视觉设计，可分享（安装商当面给用户展示，用户转发家人） |

#### 3.2.3 SEG 收益仪表盘（P1）

| 维度 | 说明 |
|------|------|
| **用户场景** | "我的储能今天帮我赚了多少？" — 长期留存的核心 hook |
| **核心指标** | ① 今日/本周/本月 SEG 售电收入 ② 高价窗口放电收益标注（"17:30-19:00 放电 3.2 kWh @ 50p = £1.60"）③ 累计总收益 ④ 自用省电金额 |
| **数据源** | Octopus Energy API（公开 API，提供 30 分钟粒度的 SEG 电价数据）；British Gas、E.ON 等零售商 API（需逐一对接） |
| **技术方案** | 定时拉取 SEG 电价 → 结合实际馈电量 → 计算每 30 分钟窗口的收入 → 日/周/月汇总 |
| **与德国 EPEX 的复用** | SEG 电价响应引擎与德国 EPEX 负电价规避本质是同一能力——**动态电价响应引擎**，区别仅在数据源和结算粒度（德国 15 分钟，英国 30 分钟） |

### 3.3 B 端安装商工具规划

#### 3.3.1 核心痛点

WHP 框架下安装商面临**三重认证负担**：

| 认证 | 适用场景 | 要求 |
|------|---------|------|
| **TrustMark** | WHP 补贴项目（WH:LG / WH:SHF） | 政府唯一认可的住宅施工质量体系，需注册并持续合规 |
| **PAS 2030:2023** | WHP 补贴项目 | 能效改造施工标准，2023 版是唯一有效版本 |
| **MCS** | 私人市场 / BUS 热泵补贴 | 可再生能源安装认证，Clean Heat Mechanism 唯一认可标准 |

安装商在 WHP 项目中的行政开销：每个项目需要提交产品合规文件 + 填写 PAS 2030 格式安装报告 + 上传至 TrustMark 系统 + 管理项目进度（12-20 周）。**小型安装商（1-5 人团队）的合规行政成本可能占项目利润的 15-25%。**

#### 3.3.2 功能规划

| 功能 | 做什么 | 怎么做 | 优先级 |
|------|--------|--------|--------|
| **合规文档包** | 按产品 SKU 一键生成 WHP 所需的全套合规文件（CE 认证、MCS 文件、技术规格书、安全测试报告、保修声明）| 预制文档模板库 → 选择型号 → 自动填充参数 → 打包下载 PDF | **P0** |
| **安装报告生成器** | 安装完成后填写关键参数 → 自动生成 PAS 2030:2023 格式报告 → 导出或上传 TrustMark | 字段：系统容量、逆变器型号、接线方式、测试数据（绝缘/接地电阻）、现场照片。输出含品牌 logo 的 PDF | **P0** |
| **EPC 数据查询** | 输入客户邮编/地址 → 查询 EPC 评级 → 判断 WHP 资格（D-G） | 调用 EPC Open Data API（免费，无需审批）。返回评级、建议改进项、证书有效期 | **P1** |
| **经济性测算工具** | 安装商上门时当场展示"安装后经济账"，作为销售辅助工具 | 与 C 端经济账计算器共用引擎，但 B 端呈现加入"与竞品对比"、"补贴后毛利"等安装商关心的维度 | **P1** |
| **项目管道管理** | 看板视图管理多个并行项目（WHP 项目周期 12-20 周）| 卡片含：客户信息、EPC 评级、补贴类型、当前阶段、下一步行动、截止日期。支持筛选排序 | **P1** |

**技术选型：Web 优先。** 安装商工作场景是"上门 → iPad/笔记本 → 填报告"，Web App 覆盖所有设备，无需下载。如果产品配网走 WiFi 而非蓝牙，无需原生 App。

### 3.4 平台层规划 — EPC Smart Readiness

2026-10 EPC 改革是战略性窗口。新 EPC 将包含四个维度：

| 新 EPC 指标 | 说明 | 储能产品的影响 |
|------------|------|--------------|
| **能源成本**（Energy Cost） | 替代当前的 SAP 评分 | 储能降低购电成本 → 提升评级 |
| **织物性能**（Fabric Performance） | 房屋隔热水平 | 与储能无直接关系 |
| **供暖系统性能**（Heating System） | 供暖效率 | 储能+热泵协同可提升 |
| **智能就绪度**（Smart Readiness） | 房屋智能化程度 | **储能是核心加分项** |

**Smart Readiness 是储能厂商能直接影响的维度。** 如果云平台能输出结构化的"智能就绪度证明"数据包（支持 SEG 分时响应 ✓、支持 VPP ✓、支持远程 OTA ✓、支持热泵协同 ✓），就可以帮助房屋获得更高的 EPC 评级——**储能从"省电工具"升级为"房屋增值资产"。**

## 四、Social Housing — 批量部署场景的软件需求

### 4.1 Social Housing 与私人住宅的本质差异

| 维度 | 私人住宅（WH:LG / 贷款） | Social Housing（WH:SHF） |
|------|------------------------|-------------------------|
| **决策者** | 户主本人 | Housing Association / 地方政府 |
| **采购模式** | 单户零售 | 批量采购（一栋楼/社区统一改造） |
| **App 用户** | 户主（C 端） | 物业管理方 + 租户（B 端 + C 端） |
| **核心关注** | 个人 ROI、电费节省 | 合规报告、批量资产管理、ESG 数据 |
| **安装模式** | 安装商一户一户做 | 总包商批量部署 |
| **电费受益方** | 户主直接受益 | 租户受益，但房东需向政府证明减碳成效 |
| **渠道链路** | 安装商推荐 → 品牌 | 公共采购招标（framework agreement）→ 品牌 |

### 4.2 三层软件需求详解

#### 4.2.1 合规报告 — "向政府证明钱花对了"

Housing Association 拿了 WH:SHF 补贴，必须向 DESNZ 定期提交数据。这是拿钱的前置条件。

**报告内容：**

| 报告维度 | 具体指标 | 数据来源 | 频率 |
|---------|---------|---------|------|
| **措施安装进度** | 每户安装了什么措施（光伏/隔热/热泵/储能）、安装日期、施工方 | 安装商提交 | 按月/按季 |
| **EPC 升级效果** | 改造前 EPC vs 改造后 EPC（目标：D/E/F/G → C 或以上） | EPC 评估师 | 改造完成后 |
| **碳减排量** | 每户改造带来的 CO₂ 年减排量（kg CO₂/年） | SAP/HEM 计算模型 | 改造完成后 |
| **资金使用** | 每户补贴金额、对应措施、是否在预算内 | 财务系统 | 按季 |
| **区域分布** | 按地区统计改造户数、措施类型分布 | 汇总 | 按季 |

**关键政策硬约束：**

| 约束 | 内容 | 时间 |
|------|------|------|
| **EPC C 强制达标** | 所有社会租赁住宅必须达到 EPC C | 2030-04-01 |
| **执行方** | 社会住房监管局（Regulator of Social Housing），作为 Decent Homes Standard Criterion D（热舒适性） | — |
| **豁免** | 单户花费 >£10,000 仍达不到 EPC C 的，可延期 10 年 | — |
| **合规证明** | 通过 SAP 和 HEM（Home Energy Model）两种方式均可 | 2026 起 HEM 框架发布 |
| **SHDF 进度** | Wave 2.1 已改造 44,500 户，接近 100% 达到 EPC C | 截至 2026-03 |

**现有痛点：** Housing Association 内部通常用 Excel + 邮件收集安装商改造数据，手动汇总后提交 DESNZ。一个中型 Housing Association 管 5,000-20,000 套房，每户 3-5 项措施，数据量大且来源分散。

#### 4.2.2 批量资产管理 — "管 2 万套房的能源设备"

Social Housing 的本质是一个机构管理海量同质化资产。改造后每套住宅都有光伏/储能/热泵等设备需要持续监控运维。

**管理需求：**

| 管理维度 | 具体需求 | 规模感 |
|---------|---------|--------|
| **设备状态监控** | 每套房的光伏发电量、储能 SOC/SOH、热泵 COP、是否离线/告警 | 2 万台设备 × 5+ 数据点 = 10 万+ 实时指标 |
| **故障检测与工单** | 设备离线/性能异常 → 自动触发维修工单 → 派发维修团队 | 2 万台设备年均故障率 3-5% = 600-1000 工单/年 |
| **批量固件升级** | OTA 推送固件更新，按区域/型号分批执行 | 不能 2 万台同时升级，需分批策略 |
| **能耗基线对比** | 改造前 vs 改造后能耗对比，证明改造有效 | 需保留改造前历史能耗数据 |
| **设备生命周期** | 保修到期提醒、预防性维护计划、更换预算预测 | 光伏 25 年、储能 10-15 年、热泵 15-20 年 |

**现有市场竞品分析：**

| 软件 | 厂商 | 核心能力 | 短板 |
|------|------|---------|------|
| **DREam** | IRT Surveys | 改造决策建模、资金申请报告导出、多数据源整合 | 偏规划侧，不做实时设备监控 |
| **Sava Intelligent Energy** | Sava | 与 Aareon/Capita/Civica/MRI 等主流 HMS 集成，EPC 数据引擎 | 偏能效评估，不做设备层运维 |
| **Parity Projects** | Cotality（已收购） | 住房存量数据库、减碳路径规划、成本分析 | 偏政策分析，不做实时监控 |
| **Home Analytics** | Energy Saving Trust | 地址级能效快照、资金申请策略 | 偏数据分析，不做设备管理 |

**关键发现：现有竞品集中在"规划侧"（怎么改造最划算），没有一家做"运维侧"（改造完了怎么管设备）。** 这个空白恰好是储能厂商的设备云平台可以填的——已经有设备实时监控基础能力，加一层"批量管理 dashboard + 工单系统 + 批量 OTA"就能切入 Social Housing 场景。

#### 4.2.3 ESG 数据 — "向投资人和监管证明可持续"

Housing Association 不只需要向政府交合规报告，还需要向投资人、评级机构、行业协会证明 ESG 表现。

**核心框架：SRS v2.1（Sustainability Reporting Standard for Social Housing）**

| 维度 | 说明 |
|------|------|
| **性质** | 社会住房行业唯一的专用 ESG 标准 |
| **发布方** | Sustainability for Housing |
| **采用规模** | 138 家住房提供商，覆盖 260 万+ 套住宅 |
| **当前版本** | v2.1，2026-10 报告周期启用 |
| **最新数据** | 104 家提供商报告中，80% 的 260 万套住宅达到 EPC C 或以上 |

**SRS v2.1 能源和气候相关指标（E 维度）：**

| 指标 | 数据来源 | 储能产品能贡献什么 |
|------|---------|-----------------|
| **EPC 评级分布**（新建+存量） | EPC 数据库 | 光伏+储能提升 EPC 评级 |
| **净零目标与策略** | 内部文件 | 设备级碳减排数据支撑净零路径 |
| **Scope 1/2/3 温室气体排放** | 能耗数据计算 | Scope 2（用电排放）通过光伏自用+储能显著降低 |
| **洪涝和过热风险** | 风险评估 | 储能备电能力可作为极端天气韧性措施 |
| **生物多样性/废弃物/水管理策略** | 内部文件 | 与储能无直接关系 |

**SRS v2.1 其他维度（S + G）：**

| 维度 | 指标示例 | 与储能关系 |
|------|---------|-----------|
| **社会（S）** | 租金水平对比、消防/燃气安全、租户满意度、福祉支持 | 储能降低租户电费 → 间接提升满意度 |
| **治理（G）** | 监管评级、董事会多样性、薪酬平等 | 无直接关系 |

**ESG 数据对储能厂商的战略意义：**

Housing Association 在选择改造方案供应商时，越来越看重供应商能否提供结构化的 ESG 数据。如果云平台能自动输出"已安装 X 套储能 → 年化减排 Y 吨 CO₂ → 租户年均省电 £Z"的数据包，Housing Association 可直接塞进 SRS v2.1 报告——**降低 ESG 报告成本 = 提高在采购清单中的评分。**

### 4.3 三层需求之间的关系

```
                     投资人 / 评级机构 / 行业协会
                              ↑
                    ③ ESG 数据（SRS v2.1）
                    "我们的住房存量有多绿？"
                              ↑
                              │ 数据汇聚
                              │
          DESNZ / 社会住房监管局              Housing Association 运维团队
                    ↑                                   ↑
        ① 合规报告（SHDF KPI）              ② 批量资产管理
        "补贴花在哪了？EPC 升了没？"         "2 万台设备有没有在正常工作？"
                    ↑                                   ↑
                    │                                   │
              ┌─────┴─────┐                    ┌────────┴────────┐
              │ 安装商交付数据│                  │ 设备云平台实时数据  │
              │ EPC 评估数据 │                  │ 告警 / 工单 / OTA │
              └───────────┘                    └─────────────────┘
```

三层是递进关系：
1. **合规报告**是底层——拿补贴必须做，否则资金被收回
2. **批量资产管理**是中间层——改造完了设备要能管，否则装了没人维护
3. **ESG 数据**是顶层——融资、评级、行业对标，管理层和董事会关心

## 六、核心结论

1. **ECO 框架终结 + WHP 接棒 = 未来 4 年英国家储补贴的唯一支撑**。不存在"等下一个政策"的选项
2. **储能在 WHP 中是"光伏配套"而非独立品类**。App 经济性计算必须是"光伏+储能"套装逻辑
3. **安装商合规成本是最大痛点，也是最大渠道绑定机会**。TrustMark + PAS 2030 的行政负担让小安装商苦不堪言
4. **2026-10 EPC 改革的 Smart Readiness 指标是被低估的战略窗口**。储能的智能能力将直接影响房屋 EPC 评级
5. **Social Housing 的运维侧软件是市场空白**。现有竞品（DREam、Sava、Parity Projects）全部偏规划侧，没有做改造后设备运维。储能厂商天然拥有这个能力，3-4.5 人月即可低成本切入
6. **SEG 分时售电 + VPP 与德国 EPEX 负电价规避是同一个软件能力**——动态电价响应引擎，跨市场复用成本低
7. **SRS v2.1 ESG 报告能力是进入 Social Housing 采购清单的软件差异化**。能自动输出碳减排数据包的厂商在招标评分中占优
